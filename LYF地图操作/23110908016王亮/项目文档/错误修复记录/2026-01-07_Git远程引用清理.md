# 错误修复记录：Git 远程引用清理 (2026-01-07)

## 1. 错误描述
**现象**：在尝试删除远程分支（如 `新功能`）时，Git 报错：
`Git: unable to delete '新功能': remote ref does not exist`

**环境**：IDE (VS Code/Visual Studio) 集成 Git 管理器。

## 2. 原因分析
该分支最初是存在的，但在远程仓库（GitHub）端可能已被手动删除或通过其他成员合并后删除。
本地 Git 仓库由于未及时执行同步操作，仍保留着对该远程分支的“追踪引用”（Remote-tracking branch）。当通过本地提交删除指令时，本地 Git 指向了一个在服务器上已失效的引用，导致服务器返回错误。

## 3. 解决方案
使用 `git fetch --prune`（或 `git remote prune <remote_name>`）来同步远程仓库的状态并清理本地过时的引用。

### 执行步骤：
1. 打开终端。
2. 运行以下命令：
   ```bash
   git fetch 团队 --prune
   ```
   *(注：本项目的远程仓库名称为 `团队`)*
3. 检查分支列表：
   ```bash
   git branch -a
   ```
   确认原本显示的 `remotes/团队/新功能` 等分支已消失。

## 4. 预防措施
- 建议在执行删除或合并操作前，先运行 `git fetch --prune` 保持引用同步。
- 团队协作中，建议开启 Git 设置：`git config --global fetch.prune true`，使其在每次 fetch 时自动清理过时分支。
